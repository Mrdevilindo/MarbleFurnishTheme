MarbleCraft Theme Structure Documentation

Key Features

  • Custom Post Type: Marble Products with price, currency, and featured options
  • Multilingual Support: Compatible with WPML and Polylang for translations
  • Contact Form: Custom form with database storage and email notifications
  • Responsive Design: Mobile-friendly layout with Tailwind CSS
  • Product Gallery: Image gallery with thumbnails for product pages

Theme Structure

marblecraft/ ├── style.css # Main stylesheet ├── functions.php # Theme functions ├── index.php # Main template ├── header.php # Header template ├── footer.php # Footer template ├── front-page.php # Homepage template ├── page.php # Default page template ├── page-products.php # Products page template ├── page-contact.php # Contact page template ├── single-marble_product.php # Single product template ├── archive-marble_product.php # Product archive template ├── template-parts/ # Template components │ ├── product-card.php # Product card component │ ├── contact-form.php # Contact form component │ └── language-switcher.php # Language switcher component ├── inc/ # Theme includes │ ├── custom-post-types.php # Custom post type definitions │ ├── form-handler.php # Form processing logic │ └── admin-submissions.php # Admin page for form submissions └── assets/ # Theme assets ├── css/ # CSS files ├── js/ # JavaScript files └── images/ # Image files

Implementation Notes

This theme was designed to work within a WordPress environment. The static demo you're seeing is just a representation of the theme's design.

WordPress Integration Requirements:

  • WordPress core files
  • MySQL database for storing content and form submissions
  • Optional: WPML or Polylang plugin for multilingual support
  • Optional: Google reCAPTCHA for form spam protection

Custom Database Tables:

CREATE TABLE wp_marblecraft_submissions ( id bigint(20) NOT NULL AUTO_INCREMENT, name varchar(255) NOT NULL, email varchar(255) NOT NULL, country varchar(100) NOT NULL, message text NOT NULL, submission_date datetime DEFAULT CURRENT_TIMESTAMP, PRIMARY KEY (id) );
